# Архитектурный анализ проекта Quantum IDE 🚀
> **Quantum IDE** — это высокотехнологичная, мобильная среда разработки (IDE) нового поколения со встроенным искусственным интеллектом, работающая непосредственно на Android.

Данный документ представляет собой **полное руководство и системный анализ кодовой базы** для разработчиков. Он подробно описывает структуру проекта, назначение каждого файла, архитектуру изолированной песочницы (Ubuntu 24.04 Sandbox) и систему ИИ.

---

## 🛠️ Стек технологий (Tech Stack)

Приложение построено на базе современной кроссплатформенной экосистемы:
*   **Фреймворк**: Flutter (версия 3.x, поддержка Material 3, кастомная отрисовка).
*   **Управление состоянием**: Riverpod (декларативный реактивный подход с разделением слоёв).
*   **Навигация**: GoRouter (модульный декларативный роутинг).
*   **Ядро Редактора**: `re_editor` + `re_highlight` (высокопроизводительный текстовый редактор с поддержкой виртуализации строк, подсветки синтаксиса и кастомных маркеров на полях).
*   **Терминал и PTY**: `xterm.dart` (рендеринг терминала) + `flutter_pty` (мост к низкоуровневым системным псевдо-терминалам через C-биндинги и FFI).
*   **Искусственный Интеллект**: `google_generative_ai` (Gemini Pro/Flash) + кастомные коннекторы к OpenAI, Anthropic, Ollama, LM Studio.
*   **Дизайн система**: `flex_color_scheme` для динамических палитр, шрифт Outfit (интерфейс) и JetBrains Mono (код), эффекты Glassmorphic (размытие заднего фона с эффектом матового стекла).

## ⚙️ Глубокий анализ ключевых сервисов (`lib/core/services/`)

Сервисный слой инкапсулирует сложные фоновые вычисления и взаимодействие с операционной системой:

1.  **`RuntimeService` (`runtime_service.dart`)**
    *   *Назначение*: Обеспечивает начальный запуск виртуальной среды разработки.
    *   *Как работает*: Распаковывает образ `Ubuntu rootfs` во внутреннее хранилище приложения, генерирует файлы `/etc/passwd`, `/etc/group` для имитации корректного root-пользователя в бесправном Android, компилирует персонализированный файл автозапуска `.bashrc` и настраивает псевдо-терминалы.

2.  **`LspService` (`lsp_service.dart`)**
    *   *Назначение*: Интеграция протокола языкового сервера (Language Server Protocol) для автодополнения кода, перехода к определению (Go to Definition) и подсветки ошибок.
    *   *Как работает*: Запускает внутри Ubuntu PRoot процесс анализатора (например, `dart language-server`) и обменивается с ним асинхронными JSON-RPC командами через стандартные потоки ввода-вывода (Stdin/Stdout).

3.  **`PackageService` (`package_service.dart`)**
    *   *Назначение*: Управление инструментами сборки и системными библиотеками.
    *   *Как работает*: Проверяет наличие компиляторов и утилит (`git`, `openjdk`, `aapt2`, `cmake`, `ndk`). В случае их отсутствия — осуществляет автоматическую загрузку, распаковку, создание символических ссылок и прописывает переменные `PATH`.

4.  **`ProjectService` (`project_service.dart`)**
    *   *Назначение*: Оркестратор сборки проектов на мобильном устройстве.
    *   *Как работает*: Вызывает команды сборки (например, `flutter build apk` или `gradlew assembleDebug`) внутри Ubuntu PRoot, перехватывает лог сборки в реальном времени и выводит его в терминал, обрабатывает финальные скомпилированные артефакты.

5.  **`AiAutocompleteService` (`ai_autocomplete_service.dart`)**
    *   *Назначение*: Наш кастомный сервис "умного" автодополнения кода на лету.
    *   *Как работает*: Слушает ввод пользователя в реальном времени. Обладает **смарт-дебаунсом в 700 мс** (чтобы не перегружать сеть и лимиты API). При остановке печати собирает до 50 строк кода до курсора и 50 строк после, отправляет контекстный запрос в Gemini, отсекает markdown-теги разметки кода и мгновенно перерисовывает всплывающее меню редактора, предлагая готовую многострочную вставку кода.

## 🏗️ Архитектура Песочницы (PRoot & Ubuntu Sandbox)

Вся компиляция и выполнение команд происходят внутри изолированного Linux-контейнера. 

*   **Как это работает**: Используется утилита **PRoot** (пользовательская реализация системного вызова `chroot` через перехват обращений к ядру с помощью `ptrace`). Это позволяет приложению без прав Root на Android эмулировать полноценную корневую файловую систему Ubuntu Linux.
*   **Скрипт Запуска (`assets/init-host.sh`)**:
    *   Монтирует системные каталоги хоста Android (`/dev`, `/proc`, `/sys`) внутрь виртуальной Ubuntu.
    *   Монтирует кастомные исполняемые файлы среды в `/usr/local/bin`.
    *   Запускает интерпретатор `/bin/bash --login` в качестве интерактивной сессии терминала.

## 🤖 Модуль Искусственного Интеллекта (AI Engine)

В Quantum IDE реализован мощный многомодельный ИИ-клиент, управляемый через `aiServiceProvider`:

*   **Локальные и облачные бэкенды**:
    *   *Cloud*: Прямое подключение к Google Gemini API (модели `gemini-2.0-flash`, `gemini-1.5-pro` и др.), OpenAI API (GPT-4o), Anthropic Claude.
    *   *Local*: Поддержка работы с локально запущенными на девайсе или сервере моделями через **Ollama** и **LM Studio** (совместимый с OpenAI JSON-формат).
*   **Режимы интеграции**:
    *   *Чат-ассистент*: Боковая/нижняя панель для свободного общения с передачей контекста текущего открытого файла.
    *   *AI Агенты*: Интерфейс запуска автономных сценариев (CLI инструменты).
    *   *AI Autocomplete*: Фоновый помощник, пишущий код параллельно с разработчиком.

## 💎 Уникальный дизайн (Premium UI/UX)

Интерфейс приложения разработан для создания вау-эффекта:
*   **Эффект Стекла (Glassmorphism)**: Использование `BackdropFilter` с размытием `blur = 20..40` поверх глубоких темных градиентов (`Color(0xFF080A10)`).
*   **Микро-анимации**: Вкладки переключаются с использованием сглаженных переходов `AnimatedContainer`, элементы дерева файлов анимированно разворачиваются с эффектом пружины.
*   **Световые акценты**: Поддержка динамической палитры `flex_color_scheme` (по умолчанию Mandy Red, с возможностью выбора глубокого синего Deep Blue, изумрудного Emerald и др.), кастомные неоновые рамки и индикаторы.

## 🔮 Расширения 2026 года (Дополнения 2026)

Для вывода Quantum IDE на уровень ультимативного инструмента разработки на 2026 год, в архитектуру закладывается расширение структуры папок и внедрение следующих прорывных модулей:

### 1. Дополнения в Сервисном Слое (`lib/core/services/`)
*   **`ai_agent_orchestrator.dart`** 🆕
    *   *Назначение*: Управление автономными действиями ИИ-агентов.
    *   *Принцип работы*: Позволяет ИИ не просто советовать код, а выполнять bash-команды в `NativeTerminal`, запускать/останавливать процессы компиляции, создавать и модифицировать файлы без участия пользователя на основе высокоуровневых задач.
*   **`crdt_sync_service.dart`** 🆕
    *   *Назначение*: Синхронизация изменений файлов для режима совместной разработки (Live Share).
    *   *Принцип работы*: Использует бесконфликтные реплицируемые типы данных (CRDT) для бесконфликтного слияния правок от нескольких удалённых разработчиков в реальном времени.
*   **`microvm_service.dart`** 🆕
    *   *Назначение*: Альтернативный слой изоляции для Android 15+.
    *   *Принцип работы*: На Android 15 накладываются жёсткие ограничения на ptrace (блокируя PRoot). Сервис `MicroVM` использует встроенный аппаратный гипервизор (KVM) для запуска сверхлёгких виртуальных машин (MicroVM) с полноценным ядром Linux, гарантируя 100% стабильность компиляции.
*   **`wasm_plugin_runner.dart`** 🆕
    *   *Назначение*: Исполнитель кастомных плагинов расширения среды.
    *   *Принцип работы*: Позволяет пользователям писать плагины на любом языке (Rust, Go, C) с последующей их компиляцией в WebAssembly (.wasm) для быстрого и безопасного выполнения внутри IDE.

### 2. Новые Функциональные Модули (`lib/features/`)
*   **`collaboration/` (Совместная разработка)** 🆕
    *   `live_session_notifier.dart`: Менеджер сессий подключения, обмена токенами и синхронизации файлов.
    *   `remote_cursor_overlay.dart`: Кастомный слой рендеринга поверх виджета `re_editor`, который отображает курсоры, выделения и имена других участников сессии в реальном времени.
*   **`plugins/` (Кастомные расширения)** 🆕
    *   `plugin_manager_page.dart`: Графический интерфейс для поиска, установки, включения и настройки WebAssembly плагинов.

### 3. Общие Адаптивные Компоненты (`lib/shared/`)
*   **`foldable_split_manager.dart`** 🆕
    *   *Назначение*: Менеджер окон для Fold-устройств и гибких экранов.
    *   *Принцип работы*: Динамически разделяет рабочую область на несколько колонок при сгибе устройства (сенсоры Fold-смартфонов), плавно распределяя редактор кода, превью браузера и терминал по физическим половинкам экрана.
